React는 페이스북에서 개발한 오픈 소스 자바스크립트 라이브러리로, 사용자 인터페이스(UI)를 구축하는 데 사용된다. 컴포넌트 기반 아키텍처를 채택하여 재사용 가능한 UI 구성 요소를 만들고 관리할 수 있게 해준다.
이번 글에서는 React를 사용하는 이유와 주요 특징 및 장점들에 대해 알아보자.

## TL;DR

- React는 복잡한 UI를 효율적으로 관리하고 개발할 수 있게 해주는 강력한 라이브러리이다.
- React는 선언적 프로그래밍, 컴포넌트 기반 아키텍처, 단방향 데이터 흐름, JSX, 가상 DOM 등의 특징을 통해 개발자의 생산성을 높이고 유지보수성을 향상시킨다.
- React는 활발한 커뮤니티와 생태계 덕분에 다양한 리소스를 활용할 수 있다.

## React가 등장하게 된 이유

웹 애플리케이션이 점점 더 복잡해지면서 사용자 인터페이스를 효율적으로 관리하는 것이 큰 도전 과제가 되었다. 전통적인 방식으로는 상태 변화에 따른 UI 업데이트가 어렵고, 코드가 복잡해지기 쉬웠다. 아래 코드는 간단한 카운터 예제이지만, 상태 업데이트와 UI 업데이트 라인이 섞여 있어 UI가 복잡해질 수록 유지보수도 어려워질 것을 예측해볼 수 있다.

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>전통적인 방식의 카운터</title>
  </head>
  <body>
    <div>
      <p id="count-display">Count: 0</p>
      <button id="increment">+</button>
      <button id="decrement">-</button>
    </div>

    <script>
      // 상태 (state)
      let count = 0;

      // UI 업데이트 함수
      function updateUI() {
        const display = document.getElementById('count-display');
        display.textContent = 'Count: ' + count;
      }

      // 이벤트 리스너 등록
      document
        .getElementById('increment')
        .addEventListener('click', function () {
          count++; // 상태 업데이트
          updateUI(); // DOM 갱신
        });

      document
        .getElementById('decrement')
        .addEventListener('click', function () {
          count--; // 상태 업데이트
          updateUI(); // DOM 갱신
        });

      // 초기 UI 렌더링
      updateUI();
    </script>
  </body>
</html>
```

페이스북은 이러한 문제를 해결하기 위해 React를 개발했다.
React는 다음과 같은 문제를 해결하기 위해 등장했다.

- **복잡한 UI 관리**: 상태 변화에 따른 UI 업데이트를 쉽게 처리하기 위해.
- **재사용성 부족**: UI 구성 요소를 재사용하고 유지보수하기 쉽게 만들기 위해.
- **개발자 경험 향상**: 선언적 프로그래밍 방식을 도입하여 개발자가 UI를 더 쉽게 이해하고 작성할 수 있도록 하기 위해.

이전에는 상태 업데이트와 UI 업데이트 라인들을 개발자가 직접 관리해야했지만, React를 사용함으로써 상태 변화에 따른 UI 업데이트를 React가 대신 처리하도록 할 수 있게 되었다.

[React 구 공식문서](https://legacy.reactjs.org/)에도 다음과 같은 내용이 적혀있다.

> React makes it painless to create interactive UIs. <u>Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes.</u>

즉, 우리가 애플리케이션에서 각 상태를 참조하는 뷰를 설계하면, 데이터가 변경될 때 React가 적절한 컴포넌트만 효율적으로 업데이트하고 렌더링해준다는 것이다.

## React의 주요 특징 및 장점

### 1. 선언적 프로그래밍

React는 선언적 프로그래밍 방식을 채택하여, 개발자가 UI가 어떻게 보여야 하는지를 선언적으로 기술할 수 있게 해준다. 이는 코드의 가독성과 유지보수성을 크게 향상시킨다.

```jsx
const Counter = () => {
  const [count, setCount] = React.useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
    </div>
  );
};
```

### 2. 컴포넌트 기반 아키텍처

React는 UI를 독립적이고 재사용 가능한 컴포넌트로 분리하여 개발할 수 있게 해준다. 각 컴포넌트는 자체 상태와 생명주기를 가지며, 이를 통해 복잡한 UI를 더 쉽게 관리할 수 있다.

```jsx
const Button = ({ onClick, children }) => {
  return <button onClick={onClick}>{children}</button>;
};

const App = () => {
  const handleClick = () => {
    alert('Button clicked!');
  };

  return (
    <div>
      <h1>Hello, React!</h1>
      <Button onClick={handleClick}>Click Me</Button>
    </div>
  );
};
```

### 3. 단방향 데이터 흐름

React는 단방향 데이터 흐름을 채택하여, 데이터가 부모 컴포넌트에서 자식 컴포넌트로만 전달되도록 한다. 이는 데이터의 흐름을 더 예측 가능하게 만들고, 디버깅을 쉽게 해준다.

```jsx
const Child = ({ message }) => {
  return <p>{message}</p>;
};

const Parent = () => {
  const [message, setMessage] = useState('Hello from Parent');
  return <Child message={message} />;
};
```

### 4. JSX (JavaScript XML)

JSX는 자바스크립트 코드 안에 HTML과 유사한 문법을 사용할 수 있게 해주는 확장문법이다. JSX를 사용하면 UI 구조를 더 직관적으로 표현할 수 있다. JSX는 Babel을 통해 결국 순수한 자바스크립트로 변환된다.

- 이전에 작성했던 [JSX to DOM](/blog/react/jsx-to-dom) 참고

### 5. 가상 DOM (Virtual DOM)

상태가 바뀔 때마다 전체 UI를 다시 그리는 선언적 모델을 가능하게 하되, 실제 DOM 조작이 비싸기 때문에 가상 DOM을 도입하여 변경 사항을 계산하고 필요한 부분만 실제 DOM에 반영하도록 했다. <br />

실제 DOM 조작을 한다는것은, 이전 글 [브라우저의 페이지 렌더링 과정과 최적화](/blog/browser-page-rendering)에서 다뤘던 것처럼 DOM 조작이 가해질 때마다 렌더트리 구성, 레이아웃 계산, 페인팅 등의 과정 등을 다시 거치게 된다는 뜻이다.

가상 DOM은 항상 최고의 성능을 보장하지는 않지만, 복잡한 UI를 개발자가 더 단순한 방식으로 다룰 수 있게 해준다.

  - [React 개발자의 딜레마: 가상 DOM vs 실제 DOM 성능](https://www.reddit.com/r/react/comments/1fb6f50/a_react_developers_dilemma_virtual_dom_vs_real/?tl=ko)

### 6. 크로스 플랫폼 지원

React는 웹 뿐만 아니라 React Native를 통해 모바일 애플리케이션 개발도 지원한다. 이를 통해 웹과 모바일에서 동일한 컴포넌트 기반 아키텍처를 사용할 수 있다.

### 7. 활발한 커뮤니티

React는 방대한 커뮤니티와 생태계를 가지고 있어, 다양한 라이브러리, 도구, 리소스가 존재한다. 이는 개발자가 문제를 해결하고 새로운 기능을 쉽게 통합할 수 있게 해준다.

[State of JS 2024](https://2024.stateofjs.com/en-US/libraries/front-end-frameworks/)에 따르면, React는 여전히 가장 인기있는 프론트엔드 라이브러리로 자리잡고 있다.

## 결론

React는 복잡한 사용자 인터페이스를 효율적으로 관리하고 개발할 수 있게 해주는 강력한 라이브러리이다. 선언적 프로그래밍, 컴포넌트 기반 아키텍처, 단방향 데이터 흐름, JSX, 가상 DOM 등의 특징을 통해 개발자의 생산성을 높이고 유지보수성을 향상시킨다. 또한 활발한 커뮤니티와 생태계 덕분에 다양한 리소스를 활용할 수 있다. 이러한 이유들로 인해 많은 개발자들이 React를 선택하여 웹 및 모바일 애플리케이션을 개발하고 있다.

다른 프레임워크와 라이브러리들도 많지만, 위에서 살펴본 바와 같이 React의 수많은 장점들로 인해 여전히 많은 프로젝트에서 매력적인 선택지로 남아있다.
